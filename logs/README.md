# logs/

## 目录简介

存放应用日志，用于调试和问题追踪。应用程序同时将日志输出到 GUI 界面和本地日志文件。

## 主要文件/子目录说明

### ylxcx-{YYYYMMDD}.log
按日期划分的日志文件：
- 文件名格式：ylxcx-{年}{月}{日}.log（例如：ylxcx-20260713.log）
- 每天自动创建新的日志文件
- 包含时间戳、操作描述和错误信息
- 日志条目格式：[HH:mm:ss] 消息内容
- 记录应用启动/关闭、代理操作、错误和监控事件

## 使用说明

用于调试和问题追踪：
- 实时问题诊断：通过查看最新日志文件了解应用行为
- 错误追踪：所有捕获的异常都会记录到日志中
- 性能分析：可以分析代理启动/停止时间和频率
- 审计追踪：记录所有用户操作和系统事件
- 日志文件可以安全地删除或归档，不会影响应用运行

## 注意事项

1. **日志文件增长**：
   - 日志文件会随着时间增长，特别是在频繁操作或错误多的情况下
   - 建议定期清理旧日志文件或实现日志轮转策略
   - 当前每天一个文件的策略已经提供了基本的轮转

2. **日志内容敏感性**：
   - 日志中可能包含代理服务器地址和端口信息
   - 目前不记录完整的用户名和密码（出于安全考虑）
   - 但仍需注意日志文件的访问权限，避免敏感信息泄露

3. **文件访问权限**：
   - 日志写入失败不会影响 GUI 显示（有容错机制）
   - 但持续的写入失败可能表明磁盘空间或权限问题
   - 建议监控日志目录的可用空间和写入权限
