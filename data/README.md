# data/

## 目录简介

存放 JSON 配置文件，用于代理配置的持久化存储。应用程序启动时从这里读取配置，退出时保存更改到这里。

## 主要文件/子目录说明

### config.json
JSON 格式的本机配置文件：
- 存储所有用户添加的代理配置
- 包含代理的所有属性：Id、名称、远程主机/端口、用户名/密码、本地主机/端口、状态、创建时间
- 格式化为易读的缩进 JSON，便于手动检查和编辑
- 应用程序启动时自动创建（如果不存在）
- 文件被 `.gitignore` 忽略，不得提交到公共仓库

### config.example.json

仓库中提交的脱敏模板，仅包含空的 `Proxies` 数组。复制或参考此文件创建本机配置，禁止在模板中写入真实代理地址、用户名或密码。

## 使用说明

代理配置持久化存储：
- 所有通过 GUI 添加的代理都会保存到此文件
- 应用程序启动时自动从这里读取配置并恢复代理列表
- 手动编辑此文件需要小心，确保保持有效的 JSON 格式
- 配置更改通过「添加代理」和「删除代理」操作自动保存
- 启动/停止代理操作只更新内存中的状态，不立即写入文件
- 只有当配置实际变化时（添加/删除代理）才会写入文件

## 注意事项

1. **文件位置**：
   - 活动配置文件固定为应用根目录下的 `data/config.json`
   - `MainViewModel` 使用 `AppSettings.json` 的 `Proxy.DataDirectory` 和 `Proxy.ConfigFileName`
   - 所有相对路径由 `PathResolver` 解析，不依赖当前工作目录或手写多级 `..`

2. **凭据与并发访问风险**：
- 用户名和密码由 Windows DPAPI 按当前用户加密后写入；不要复制密文到其他用户或机器
   - 当前实现没有文件锁定机制
   - 理论上如果多个实例同时运行可能导致冲突
   - 生产环境考虑添加文件锁或切换到支持并发的数据库（如 SQLite）

3. **数据完整性**：
   - JSON 解析失败时会回退到空配置而不是崩溃
   - 建议定期备份此文件以防意外损坏
   - 考虑添加配置版本控制以支持未来 schema 变更
