## 环境配置

### 1️⃣ 安装 .NET SDK

**方式 A：官方安装**

```powershell
# 查看已安装版本
dotnet --version

# 如需安装，访问 https://dotnet.microsoft.com/download
```

**方式 B：使用 Visual Studio 安装器**

- 打开 Visual Studio Installer
- 选择"修改" → 勾选".NET 10.0 SDK"
- 安装

### 2️⃣ 克隆项目

```powershell
git clone https://github.com/your-org/YLproxy.git
cd YLproxy
```

### 3️⃣ 初始化开发环境

```powershell
# 安装 Git pre-commit 钩子（防止敏感文件被提交）
pwsh -NoProfile -ExecutionPolicy Bypass -File .\scripts\init-environment.ps1 -InstallHooks

# （可选）注册日志清理计划任务（需要管理员权限）
# pwsh -NoProfile -ExecutionPolicy Bypass -File .\scripts\init-environment.ps1 -InstallScheduledTask
```

### 4️⃣ 验证依赖

```powershell
# 检查 .NET 版本
dotnet --version

# 恢复 NuGet 包
dotnet restore
```

## 编译运行

### 方式 1：命令行编译（推荐新手）

```powershell
# 进入项目根目录
cd e:\GZQ\YLXCX\YLproxy

# 清理旧构建
dotnet clean

# 还原包
dotnet restore

# 编译整个解决方案
dotnet build

# 验证编译成功
echo "编译完成！错误数：0"
```

### 方式 2：Visual Studio 编译（推荐）

1. 打开 `YLproxy.sln`
2. 右键解决方案 → "生成解决方案"（Ctrl + Shift + B）
3. 输出窗口显示 `Build succeeded`

### 方式 3：运行 GUI 程序

```powershell
# 构建 GUI 项目
dotnet build src/YLproxy.GUI/YLproxy.GUI.csproj

# 运行程序
dotnet run --project src/YLproxy.GUI/YLproxy.GUI.csproj
```

## 常见问题

### Q1：编译失败，提示找不到 NuGet 包

```powershell
# 解决：清理 NuGet 缓存并重新还原
dotnet nuget locals all --clear
dotnet restore --no-cache
dotnet build
```

### Q2：.NET SDK 版本不匹配

```powershell
# 查看项目要求的版本
Get-Content "global.json"

# 查看本地安装的版本
dotnet --list-sdks

# 如需安装特定版本
# 访问 https://dotnet.microsoft.com/download
```

### Q3：运行 GUI 程序显示窗口后立即崩溃

```powershell
# 查看日志
Get-Content "logs/runtime.log" -Tail 20

# 检查配置文件格式
Get-Content "data/config.json"

# 详见 [故障排查](06-运维部署/故障排查.md)
```

### Q4：某个 csproj 打不开

```powershell
# 检查 XML 格式
[xml]$proj = Get-Content "src/YLproxy.Core/YLproxy.Core.csproj"
$proj  # 应该输出有效的 XML

# 重新加载项目（VS Code）
# 1. File → Close Folder
# 2. File → Open Folder
```

### Q5：如何在 Visual Studio Code 中调试

1. 安装扩展：`C# Dev Kit`
2. 打开项目根目录（`YLproxy`）
3. 创建 `.vscode/launch.json`（参考后续文档）
4. F5 启动调试
