## 快速开始

### 安装与启动

**第 1 步：启动应用**
```
双击运行 YLproxy.exe
或从命令行：cd C:\YLproxy && .\YLproxy.exe
```

**第 2 步：等待界面加载**
- 界面加载需要 2-5 秒
- 查看左下角的状态栏（"就绪" 或 "加载中..."）

**第 3 步：查看主界面**
- 代理列表（左侧）
- 操作按钮（上方）
- 详情面板（右侧）
- 日志查看器（下方）

### 首次使用

```
如果是首次启动，按以下步骤操作：

1. 【添加代理】按钮 → 填写代理信息
2. 【测试连通性】→ 验证代理是否可用
3. 【启动代理】→ 开始使用
4. 【监控】选项卡 → 实时查看运行状态
```

## 基础操作

### 操作 1：添加代理

**方式 A：使用按钮**
1. 点击顶部【新增】按钮或按 Ctrl+N
2. 弹出"添加代理"对话框

**方式 B：右键菜单**
1. 在代理列表空白处右键
2. 选择"新增代理"

**填写表单**

```
┌─────────────────────────────────────┐
│  添加代理                            │
├─────────────────────────────────────┤
│                                      │
│ 代理名称:  [________________]        │
│            (用于标识，例如"美国代理1") │
│                                      │
│ 代理类型:  [SOCKS5 ▼]              │
│            选项: SOCKS4, SOCKS5, HTTP│
│                                      │
│ 服务器地址: [________________]       │
│             (IP 地址或域名)           │
│                                      │
│ 端口号:    [______]                 │
│            (1-65535)                 │
│                                      │
│ 用户名:    [________________] (可选)  │
│                                      │
│ 密码:      [________________] (可选)  │
│            ☑ 显示密码               │
│                                      │
│ 备注:      [____________________]   │
│                                      │
│           【测试连通性】【保存】【取消】│
└─────────────────────────────────────┘
```

**填写示例**

```
代理名称: 美国代理1
代理类型: SOCKS5
服务器地址: 192.168.1.100
端口号: 9999
用户名: user123
密码: pass123
备注: 用于美国地区访问
```

**重要提示**
- ✅ 地址和端口必填
- ✅ 建议先测试连通性再保存
- ✅ 敏感信息会加密存储
- ❌ 不要输入明显错误的信息

1. 在代理列表中选中要测试的代理

### 操作 3：启动代理

**前置条件**
- ✅ 代理已添加
- ✅ 代理已测试成功
- ✅ 端口未被占用

**启动步骤**

1. 在代理列表中选中代理
2. 点击【启动】按钮或按 Ctrl+S
3. 观察状态栏变化（从"灰色" → "绿色"）

**启动后的验证**

```powershell
# 1. 检查界面
   - 代理图标变为绿色 ✓
   - 详情面板显示"状态: 正常"
   
# 2. 检查日志
   - 日志查看器显示 "[INFO] 代理已启动"
   
# 3. 检查端口（可选）
   netstat -ano | Select-String "9999"
   应该显示: TCP 127.0.0.1:9999 LISTENING
```

### 操作 4：停止代理

**停止步骤**

1. 在代理列表中选中代理
2. 点击【停止】按钮或按 Ctrl+Q
3. 状态从"绿色" → "灰色"

**停止后的验证**

```
- 代理图标变为灰色
- 详情面板显示"状态: 已停止"
- 日志显示 "[INFO] 代理已停止"
```

### 操作 5：删除代理

**删除步骤**

1. 在代理列表中选中代理
2. 点击【删除】按钮或按 Del
3. 确认删除提示

- ⚠️ 删除前会停止该代理
- ✅ 可从备份恢复（参考设置）

## 代理管理

### 编辑代理信息

**步骤**

1. 选中代理
2. 点击右键选择"编辑"或双击
3. 在对话框中修改信息
4. 点击"保存"

**可编辑字段**
- 代理名称
- 服务器地址（需先停止代理）
- 端口号（需先停止代理）
- 用户名和密码
- 备注

**不可编辑字段**
- 代理类型（需删除后重新添加）
- 创建时间

### 批量操作

**启动所有代理**
1. 点击【启动】按钮（不选中任何代理）
2. 系统将启动所有已停止的代理
3. 跳过已运行的代理

**停止所有代理**
1. 点击【停止】按钮（不选中任何代理）
2. 系统将停止所有运行中的代理

**测试所有代理**
1. 点击【测试】按钮（不选中任何代理）
2. 系统逐个测试所有代理
3. 显示统计结果

### 代理排序和筛选

**排序**
- 点击列标题（名称、类型、状态、连接数）
- 支持升序/降序

**筛选**
- 右侧选项卡：【全部】【活跃】【不活跃】【监控】
- 使用搜索框快速查找代理

**导出**
- 右键选择"导出为 CSV" → 保存代理列表
- 用于备份或数据分析

## 常见场景

### 场景 1：如何从 HTTP 代理切换到 SOCKS5

**背景**：之前使用 HTTP 代理，现在需要切换到 SOCKS5 代理

**步骤**
1. 删除旧的 HTTP 代理
2. 点击【新增】添加 SOCKS5 代理
3. 填写 SOCKS5 代理信息
4. 测试连通性
5. 启动代理

### 场景 2：如何管理多个代理池

**背景**：需要管理来自多个国家的代理

**方案**
```
代理分类:
├─ 美国代理
│  ├─ 美国代理1 (纽约)
│  ├─ 美国代理2 (洛杉矶)
│  └─ 美国代理3 (芝加哥)
├─ 欧洲代理
│  ├─ 英国代理1
│  ├─ 法国代理1
│  └─ 德国代理1
└─ 亚洲代理
   ├─ 日本代理1
   ├─ 新加坡代理1
   └─ 印度代理1
```

**操作**
1. 使用代理名称前缀进行分类（例如"US-"、"EU-"）
2. 使用筛选功能快速查看特定地区代理
3. 使用备注字段记录更多信息

### 场景 3：如何监控代理健康状态

**背景**：需要实时监控多个代理的可用性

**步骤**
1. 启动所有代理
2. 打开【监控】选项卡
3. 查看每个代理的连接数、延迟等指标
4. 定期测试代理连通性
5. 根据日志分析异常

**建议**
- 每小时测试一次所有代理
- 设置告警阈值（例如：失败 3 次自动禁用）
- 定期导出日志进行分析

### 场景 4：如何处理代理连接超时

**背景**：某个代理经常超时，影响业务

**诊断步骤**
1. 选中超时的代理
2. 点击【测试】测试连通性
3. 查看日志找到超时的确切时间
4. 检查网络连接是否正常
5. 尝试修改超时设置

**解决方案**
- A. 检查代理服务器是否在线
- B. 尝试更换代理或更新密码
- C. 检查防火墙设置
- D. 在设置中增加超时时间

## 故障处理

### 故障 1：代理启动失败，报错"端口被占用"

**症状**：启动代理时弹出错误提示

**原因**
- 该端口已被其他应用占用
- 上一次关闭不彻底，进程仍在运行
- 系统防火墙阻止

**解决方案**

```powershell
# 方案 A：更换端口
1. 删除该代理
2. 重新添加，使用不同的端口（例如 10000）
3. 重新测试和启动

# 方案 B：释放端口（高级）
# 1. 打开 PowerShell (管理员)
# 2. 查找占用端口的进程
   netstat -ano | Select-String ":9999"
# 3. 结果示例：TCP 127.0.0.1:9999 LISTENING 1234
# 4. 查看进程信息
   Get-Process -Id 1234
# 5. 如果是旧的 YLproxy 进程，关闭它
   Stop-Process -Id 1234 -Force
# 6. 重新启动 YLproxy 代理

# 方案 C：运行权限
1. 以管理员身份运行 YLproxy
2. 重新尝试启动代理
```

### 故障 2：代理测试失败，报错"连接超时"

**症状**：测试代理时显示"连接超时"

**原因**
- 代理服务器离线或无响应
- 网络连接问题
- 代理地址或端口错误
- 防火墙阻止连接

**诊断步骤**

```powershell
# 1. 检查代理信息是否正确
#    确认地址和端口没有拼写错误

# 2. 检查网络连接
#    ping 代理服务器地址
ping 192.168.1.100

# 3. 检查是否可以访问该端口
#    使用 Test-NetConnection
Test-NetConnection -ComputerName 192.168.1.100 -Port 9999

# 4. 查看防火墙规则
#    检查防火墙是否允许该端口
netsh advfirewall firewall show rule name=all | Select-String "9999"

# 5. 查看代理日志
#    查看代理服务器日志了解失败原因
```

**解决方案**
- A. 检查代理服务器是否在线
- B. 验证地址和端口无误
- C. 检查网络连接和防火墙
- D. 增加超时时间（设置 → 性能 → 超时）
- E. 联系代理服务商获取帮助

### 故障 3：代理启动后无法使用，提示"连接失败"

**症状**：代理已启动（绿色），但应用无法通过代理连接

**原因**
- 代理配置错误
- 3proxy 引擎崩溃或无响应
- 操作系统防火墙阻止
- 代理地址/端口变化

**诊断步骤**

```
1. 重新测试代理连通性
   - 点击【测试】按钮
   - 如果测试成功，代理服务是正常的

2. 检查是否选择了正确的代理协议
   - 确认应用支持该代理协议
   - 例如：应用需要 SOCKS5，但配置了 HTTP

3. 检查应用代理配置
   - 确保应用指向 127.0.0.1:代理端口
   - 例如：127.0.0.1:9999

4. 查看日志找到错误
   - 在日志查看器中搜索 "ERROR"
   - 导出日志进行详细分析

5. 尝试重启代理
   - 点击【停止】
   - 等待 2 秒
   - 点击【启动】
```

**解决方案**
- A. 重新启动 YLproxy
- B. 重启应用
- C. 检查代理设置是否正确
- D. 在设置中启用调试日志获取更多信息
- E. 联系技术支持
