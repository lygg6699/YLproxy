## 常见问题

### 问题 1：程序启动时崩溃

**症状**：启动 YLproxy.exe 后立即退出
Get-Content "logs/log_$(Get-Date -Format yyyyMMdd).txt" | Select-String "3proxy"
**排查**：
```powershell
# 1. 检查配置文件
Get-Item data/config.json
Get-Item AppSettings.json
Get-Content "logs/log_$(Get-Date -Format yyyyMMdd).txt" | Select-String "Failed|ERROR|exception"
# 2. 查看日志
Get-Content "logs/log_$(Get-Date -Format yyyyMMdd).txt" -Tail 50

# 3. 检查 .NET 运行时
dotnet --version
├── log_yyyyMMdd.txt

**常见原因**：
- ❌ 配置文件丢失或格式错误
- ❌ .NET 运行时未安装
- ❌ 权限不足

Get-Content "logs/log_$(Get-Date -Format yyyyMMdd).txt" -Tail 100
```powershell
# 删除配置文件，重新生成
Remove-Item data/config.json
# 重新启动程序
Select-String "ERROR" "logs/log_$(Get-Date -Format yyyyMMdd).txt"
# 或检查权限
icacls data/
```

Select-String "3proxy-1" "runtime/3proxy/logs/3proxy-1.log"

### 问题 2：添加代理后无法测试

**症状**：点击"测试"无反应或提示超时
Get-Content "logs/log_$(Get-Date -Format yyyyMMdd).txt" -Tail 10 -Wait
**原因可能**：
- ❌ 上游代理 IP 或端口错误
- ❌ 认证信息错误
- ❌ 网络连接问题
- ❌ 防火墙阻止

Remove-Item "logs/log_$(Get-Date -Format yyyyMMdd).txt"
```powershell
# 1. 测试网络连通性
Test-NetConnection -ComputerName 123.123.123.123 -Port 8000
logs/log_yyyyMMdd.txt
# 2. 检查代理 IP 和端口是否正确
# 使用 telnet（若可用）
   - 日志文件：当前 `logs/log_yyyyMMdd.txt` 和对应的 `runtime/3proxy/logs/3proxy-{id}.log`

# 3. 验证认证信息
# 使用 curl 测试
curl -x 123.123.123.123:8000 -U username:password http://www.baidu.com
```

**解决方案**：
1. 确认代理 IP 和端口正确
2. 确认用户名和密码正确
3. 检查防火墙是否阻止连接

### 问题 3：启动代理后仍无法连接

**症状**：代理状态显示"Running"，但模拟器仍无法连接

**原因可能**：
- ❌ 本地端口监听失败
- ❌ 模拟器填写的 IP 错误
- ❌ 防火墙阻止本地端口

**排查**：
```powershell
# 1. 检查本地端口是否监听
netstat -ano | findstr :9001

# 2. 查看是否有 3proxy 进程
Get-Process | Where-Object { $_.Name -like "*3proxy*" }

# 3. 测试本地连接
telnet 127.0.0.1 9001
telnet 192.168.1.100 9001

# 4. 查看 3proxy 日志
Get-Content "runtime/3proxy/logs/3proxy-1.log" | Select-String "3proxy"
```

**解决**：
1. 确认本机 IP（`ipconfig`）
2. 在模拟器中填写正确的本机 IP
3. 检查防火墙允许本地端口

---

### 问题 4：代理运行一段时间后停止

**症状**：代理运行一会儿后，状态变为"Failed"

**原因**：
- ❌ 3proxy 进程异常退出
- ❌ 内存不足
- ❌ 配置文件错误

**排查**：
```powershell
# 查看详细日志
Get-Content "logs/log_$(Get-Date -Format yyyyMMdd).txt" | Select-String "Failed|ERROR|exception"

# 查看系统资源
Get-Process -Name 3proxy | Select-Object -Property WS, CPU

# 检查磁盘空间
Get-Volume | Where-Object { $_.DriveLetter -eq 'C' } | Select-Object Size, SizeRemaining
```

**恢复**：
1. 点击"启动"重新启动代理
2. 或删除并重新添加代理

## 日志分析

### 日志位置

```
logs/
├── log_yyyyMMdd.txt     ← 应用日志
└── runtime/3proxy/logs/3proxy-{id}.log ← 3proxy 引擎日志
```

### 日志等级

| 等级 | 说明 | 示例 |
|---|---|---|
| **DEBUG** | 调试信息 | "Deserializing config..." |
| **INFO** | 正常信息 | "Proxy started successfully" |
| **WARN** | 警告信息 | "Port already in use" |
| **ERROR** | 错误信息 | "Failed to start 3proxy" |

### 查看日志

```powershell
# 查看最后 100 行
Get-Content "logs/log_$(Get-Date -Format yyyyMMdd).txt" -Tail 100

# 查看错误日志
Select-String "ERROR" "logs/log_$(Get-Date -Format yyyyMMdd).txt"

# 查看特定代理的日志
Select-String "3proxy-1" "runtime/3proxy/logs/3proxy-1.log"

# 实时监控日志
Get-Content "logs/log_$(Get-Date -Format yyyyMMdd).txt" -Tail 10 -Wait
```

### 日志示例

```log
[2026-07-15 10:30:25.123] [INFO] [Main] YLproxy started
[2026-07-15 10:30:26.456] [INFO] [ProxyDataService] Loaded 2 proxies from config

## 恢复方法

### 快速恢复

```powershell
# 1. 停止所有 3proxy 进程
Stop-Process -Name 3proxy -Force -ErrorAction SilentlyContinue

# 2. 清除临时文件
Remove-Item runtime/3proxy/cfg/*.cfg -Force
Remove-Item runtime/3proxy/logs/* -Force

# 3. 重新启动应用
.\YLproxy.exe
```

### 完全恢复

```powershell
# 1. 删除配置文件（会丢失所有设置）
Remove-Item data/config.json

# 2. 删除日志
Remove-Item "logs/log_$(Get-Date -Format yyyyMMdd).txt"

# 3. 重新启动应用
.\YLproxy.exe
```

### 从备份恢复

```powershell
# 如有备份，可以恢复
Copy-Item data/config.json.bak data/config.json -Force

# 重新启动
.\YLproxy.exe
```
