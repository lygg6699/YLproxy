YLXCX 项目开发核心注意事项 + 完整运行逻辑。

一、开发核心原则
1. 不改变系统网络环境（最高优先级）

这是整个项目最重要的原则。

程序不能：

❌ 修改 Windows 系统代理

例如：

Windows 设置 → 网络和 Internet → 代理

不能被修改。

❌ 修改网卡配置

❌ 修改 DNS

❌ 修改路由表

❌ 创建 VPN/TUN 网络

❌ 接管系统所有流量

正确逻辑：

只有连接指定本地端口的软件
才经过代理

例如：

雷电：

192.168.1.100:9001

走代理。

浏览器：

默认网络

不受影响。

二、本地代理端口隔离设计

每个代理必须独立。

密码A

9001

账号B
密码B

 |
代理B

不能出现：

9001
 |
 |
代理A + 代理B

否则无法保证模拟器独立。

项目最核心的三个原则：
端口一一对应：一个本地端口 = 一个购买代理。
网络隔离：不碰系统网络，只服务主动填写端口的软件。
状态可控：任何代理都有明确状态，可以启动、停止、检测、恢复。

只要围绕这三个原则开发，YLXCX 的核心功能就不会偏离。
