四、API通信方案
方案选择

推荐：

本地 HTTP API

原因：

简单、稳定、调试方便。

通信：

WPF GUI

↓

http://127.0.0.1:17890/api

↓

Backend Service
五、API端口设计

固定：

127.0.0.1:17890

只允许本机访问。

例如：

http://127.0.0.1:17890/api/proxy/list

这个方案符合你的核心需求：

✅ 一个 EXE 双击运行
✅ GUI管理
✅ 输入 HTTP IP+端口+账号密码
✅ 自动生成本地端口
✅ 每个端口独立代理
✅ 不修改系统网络
✅ 模拟器按端口使用
✅ UI实时显示状态

并且架构不会过度复杂，适合个人开发和长期维护。
