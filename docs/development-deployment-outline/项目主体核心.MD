YLXCX 本地代理转换管理程序
项目核心开发说明

版本：V1.0

一、项目定位
1.1 软件类型

Windows 桌面 GUI 程序。

用户：

双击 EXE 文件启动程序。
无需打开命令行。
无需手动修改配置文件。
无需了解 3proxy。

程序负责：

接收用户输入的 HTTP 代理信息。
自动创建本地代理入口。
管理本地代理端口状态。
显示运行状态。
二、核心功能目标

软件打开后，显示一个完整的图形界面。

界面主要展示：

本机信息区域

网络类型：

8000

四、自动生成本地代理端口

程序自动创建：

例如：

输入：

123.123.123.123:8000
用户名:test001
密码:123456

自动生成：

本地代理：

127.0.0.1:9001

第二个代理：

输入：

111.111.111.111:9000

生成：

127.0.0.1:9002

对应关系：

本地9001
        ↓
第三方代理1


本地9002
        ↓
第三方代理2

六、本地端口隔离原则（核心要求）

这是项目的重要设计要求。

生成的本地端口：

例如：

127.0.0.1:9001
127.0.0.1:9002
127.0.0.1:9003

必须满足：

1. 不改变 Windows 全局网络

不能：

修改系统代理
修改浏览器代理
修改网络适配器
修改系统路由
2. 不影响其他程序

例如：

电脑上的：

浏览器
微信
游戏
下载工具

全部保持原来的网络。

3. 只有主动填写端口的软件使用代理

例如：

雷电模拟器：

填写：

IP:

192.168.1.100


端口:

9001

那么：

只有雷电模拟器通过：

9001

访问网络。

其他软件：

不会经过代理。
