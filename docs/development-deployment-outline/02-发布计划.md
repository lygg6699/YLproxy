## V1.0 完成标准

以下标准全部达成后，标记为 **v1.0.0** 稳定版：

### ✅ 基础运维

- [x] 双击 EXE 直接打开，零配置启动
- [ ] 安装包制作完成，支持一键安装 **← Phase 14**
- [ ] 支持开机自启选项 **← Phase 14**

### ✅ 网络隔离（核心约束）

- [x] 不修改 Windows 系统代理设置
- [x] 不修改网络适配器、系统路由、DNS
- [x] 不影响浏览器、微信、游戏等其他程序网络
- [x] 只有主动填写本地端口的软件经过代理

### ✅ 代理功能

- [x] 输入 HTTP 代理 IP + 端口 + 用户名 + 密码
- [x] 自动生成本地无认证端口
- [x] 每个本地端口独立对应一个上游代理
- [x] 本地端口范围：9001~9100
- [x] 支持代理连通性测试（含延迟测量）
- [x] 支持独立启动 / 停止单个代理

### ✅ 监控与状态

- [x] 后台 5 秒轮询，自动检测代理进程状态
- [x] 进程异常退出自动标记为 Failed
- [x] GUI 实时显示代理状态（Running / Stopped / Failed）

### ✅ 数据持久化

- [x] 代理配置保存到 config.json，重启不丢失
- [ ] 账号密码字段通过 DPAPI 加密存储 **← Phase 8**

### ✅ 图形界面

- [x] 显示本机域名、IP、网络状态、当前时间
- [x] 代理列表 DataGrid 展示完整信息
- [x] 添加代理弹窗（含表单校验）
- [x] 运行日志实时输出

## 更新策略

### 自动检查

```csharp
// 程序启动时自动检查更新
public async Task CheckForUpdatesAsync()
{
    var latestVersion = await GetLatestVersionFromGitHub();
    if (latestVersion > CurrentVersion)
    {
        NotifyUserToUpdate();
    }
}
```

### 升级流程

1. **检测**: 后台周期性检测最新版本（GitHub Releases API）
2. **通知**: 用户界面提示有新版本可用
3. **下载**: 下载新版本 EXE 到临时目录
4. **校验**: 验证下载文件的哈希值
5. **安装**: 执行更新程序，替换旧版本
6. **重启**: 更新完成后提示重启应用

### 回滚机制

- 更新前自动备份当前版本：`backup/YLproxy_v0.2.0.exe`
- 更新失败时自动恢复备份版本
- 用户可手动选择回滚到上一版本

### 版本锁定

用户可在设置中：
- [ ] 禁用自动检查更新
- [ ] 手动指定检查频率（每天/每周/每月）
- [ ] 选择"仅稳定版"或"包含 RC 版"
