十二、UI核心原则

这个软件的 UI 不应该像“网络工具”，而应该像：

一个简单的设备管理器。

用户看到的永远是：

输入代理

↓

看到本地端口

↓

复制端口

↓

填入模拟器

不要让用户看到：

cfg
3proxy
命令行
服务进程
配置文件

这些全部隐藏。

最终 UI 核心结构：

YLXCX
│
├── 首页
│    ├── 本机信息
│    ├── 网络状态
│    └── 代理统计
│
├── 代理管理（核心）
│    ├── 添加代理
│    ├── 自动生成端口
│    ├── 状态显示
│    └── 模拟器连接信息
│
├── 状态
│    └── 运行监控
│
├── 日志
│    └── 操作记录
│
└── 设置
     └── 基础参数

这个 UI 设计完全围绕你的核心目标：本地代理端口生成与模拟器使用，不会偏离成 VPN、系统代理或者复杂代理平台。
