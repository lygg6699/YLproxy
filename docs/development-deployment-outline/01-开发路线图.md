## 剩余开发路线

### Phase 8 — 安全加固

**目标：** 保护敏感数据（账号密码），防止明文泄露。

- [ ] 引入 Windows DPAPI 对 config.json 中的密码字段加密存储
- [ ] 数据加载时自动解密，内存使用时才明文
- [ ] 配置迁移：旧明文配置自动升级为加密格式

### Phase 9 — 存储升级（SQLite 迁移）

**目标：** 从 JSON 文件切换到 SQLite 数据库，提升并发读写可靠性与查询能力。

- [ ] 引入 SQLite NuGet 包
- [ ] 设计数据库表结构（代理配置、日志、统计）
- [ ] 实现 Repository 模式替代 ConfigService
- [ ] 数据迁移工具（json → sqlite）

### Phase 10 — UI 增强

**目标：** 更现代的视觉体验，更流畅的用户交互。

- [ ] 暗色主题支持
- [ ] 状态卡片（替代纯文本列表）
- [ ] 防重复点击保护
- [ ] 系统托盘运行（最小化到托盘）
- [ ] 日志自动滚动 + 日志级别颜色

### Phase 11 — REST API 层

**目标：** 为远程管理和外部集成提供 HTTP 接口。

- [ ] 嵌入轻量 HTTP Server（Kestrel / HttpListener）
- [ ] 端点设计：启动/停止/测试/列表/状态
- [ ] 可选的本地绑定与访问控制

### Phase 12 — 高级代理管理

**目标：** 批量操作、导入导出、代理分组。

- [ ] 批量导入/导出（JSON/CSV）
- [ ] 启动全部 / 停止全部
- [ ] 代理分组（按用途/标签）
- [ ] 代理名称搜索与过滤

### Phase 13 — 统计与日志

**目标：** 流量统计、连接记录、异常告警。

- [ ] 总流量统计（上传/下载）
- [ ] 连接日志查询
- [ ] 异常聚合告警

### Phase 14 — 自动更新与安装部署

**目标：** 一键安装、自动升级、静默部署。

- [ ] 安装包制作（Inno Setup / Squirrel）
- [ ] 自动更新检查（GitHub Releases / 自建更新服务器）
- [ ] 开机自启选项
