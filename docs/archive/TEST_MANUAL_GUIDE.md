# Task 7.8 测试指南 - 代理进程异常终止检测

## 测试概述
**目标**：验证代理进程在异常终止时，系统能否在 5 秒内正确更新状态为 *Failed*。

---

## 当前应用状态
✓ 应用已启动  
✓ 配置文件已加载  
✓ 两个测试代理已配置：
  - **Test Proxy 1**: 127.0.0.1:9001 → 107.150.105.8:998
  - **Test Proxy 2**: 127.0.0.1:9002 → 107.150.105.8:1309

---

## 操作步骤

### 步骤1：确认应用窗口显示
- [ ] 应用主窗口已打开（标题：YLproxy）
- [ ] 代理列表（DataGrid）显示两个代理：
  - Row 1: "Test Proxy 1" | 127.0.0.1 | 9001 | 107.150.105.8 | 998 | **Status: Stopped**
  - Row 2: "Test Proxy 2" | 127.0.0.1 | 9002 | 107.150.105.8 | 1309 | **Status: Stopped**

### 步骤2：启动第一个代理
- [ ] 在代理列表中点击第一行（Test Proxy 1）进行选择
- [ ] 点击"启动"按钮（Start Button）
- [ ] 观察：应用日志中应显示"Started proxy 1"
- [ ] 确认：Status 列显示 **"Running"** 并呈绿色

### 步骤3：启动第二个代理
- [ ] 在代理列表中点击第二行（Test Proxy 2）进行选择
- [ ] 点击"启动"按钮（Start Button）
- [ ] 观察：应用日志中应显示"Started proxy 2"
- [ ] 确认：Status 列显示 **"Running"** 并呈绿色

### 步骤4：验证 3proxy 进程运行
- [ ] 打开任务管理器（Ctrl + Shift + Esc）
- [ ] 在"进程"选项卡中搜索 "3proxy.exe"
- [ ] 确认看到 **2个** 3proxy.exe 进程在运行

### 步骤5：准备异常终止测试
**准备阶段**
- [ ] 记录当前时间作为 T0（进程杀死时间）
- [ ] 打开记事本或手机秒表，准备记录时间
- [ ] 确保你能清楚看到应用窗口和任务管理器

**关键测试说明**
- 在下一步中，将强制杀死所有 3proxy.exe 进程
- 应用中的 MonitorService 每 5 秒检查一次进程状态
- 系统应在 **5 秒内** 检测到进程异常终止
- 代理状态应该从 "Running" 变为 "Failed"（红色）

### 步骤6：强制终止代理进程
- [ ] 在任务管理器中选择第一个 3proxy.exe
- [ ] 点击"结束任务"按钮
- [ ] 在任务管理器中选择第二个 3proxy.exe  
- [ ] 点击"结束任务"按钮
- [ ] **记录杀死时间：T0 = __________ (HH:mm:ss.fff)**

### 步骤7：观察状态更新
**计时观察**
- T+0 秒：进程已被杀死，状态仍为 "Running"
- T+1-4 秒：继续观察，应用检查进程状态
- T+5 秒：MonitorService 完成第一个监控周期
  - [ ] 观察代理1状态是否变为 "Failed"（红色）
  - [ ] 观察代理2状态是否变为 "Failed"（红色）
  - [ ] 检查应用日志中是否显示 "process exited unexpectedly" 消息

**记录更新时间**
- [ ] 状态改变时，立即记录时间：T1 = __________ (HH:mm:ss.fff)

### 步骤8：计算检测延迟
**手工计算**
```
检测延迟 = T1 - T0
示例：T0 = 14:30:45.123, T1 = 14:30:48.456
延迟 = 3.333 秒
```

- [ ] 计算延迟时间：__________ 秒

**验证结果**
- [ ] 如果延迟 ≤ 5 秒：✓ **测试通过**
- [ ] 如果延迟 > 5 秒：✗ **测试不通过**

### 步骤9：验证日志信息
- [ ] 打开应用下方的"运行日志"ListBox
- [ ] 查找以下日志条目：
  ```
  [HH:mm:ss] Monitor: proxy 1 (127.0.0.1:9001) process exited unexpectedly
  [HH:mm:ss] Monitor: proxy 2 (127.0.0.1:9002) process exited unexpectedly
  ```

### 步骤10：清理代理配置
**删除测试代理**
- [ ] 在代理列表中选择 "Test Proxy 1"
- [ ] 点击"删除"按钮（Remove Button）
- [ ] 确认代理1已被删除
- [ ] 重复操作删除 "Test Proxy 2"
- [ ] 代理列表应显示为空
