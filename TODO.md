- [ ] 按“添加 → 保存 → 测试 → 启动 → 本地端口访问 → 停止 → 删除”完成单代理闭环。

## 四、执行顺序与依赖

1. 先完成本次清理和 P0 基线复核，确认当前版本真实可运行。
2. P0 通过后立即进入 P1 DPAPI；在 P1 完成前不建议对外发布包含真实代理凭据的版本。
3. P2 与 P1 并行推进，但任何日志/异常重构都必须保留 P0 的运行链可观测性。
4. P3、P4 依赖稳定的 Core 服务边界；P5 依赖 P0-P2 的验证和回滚材料。
5. 每完成一个阶段，同步更新 `docs/progress.md`、`docs/task-tracking.md`、`docs/deployment.md` 和 `docs/changelog.md`。
